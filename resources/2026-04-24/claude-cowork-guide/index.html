💡這是什麼，跟 Chat 差在哪
Chat 是「一問一答」，Cowork 是「一次交派整個任務」。你告訴 Claude 要做什麼，它自己規劃步驟、讀本機檔案、打開 Google Drive、寫 Email 草稿、跑完整個流程，你在旁邊審核就好。2026 年 1 月底推出 research preview，4 月正式 GA，macOS + Windows 都能用。定位是給非工程師的 Claude Code，研究員、分析師、營運、法務、財務都適用。
🌱基礎上手：5 步驟啟用 Cowork
前置條件：Claude Pro $20/月 以上方案（免費方案不支援）、macOS 或 Windows 桌面版 Claude Desktop。Computer Use 功能目前僅 macOS 可用，Windows 只支援 Connectors 和 Scheduled Tasks。
1

下載 Claude Desktop

到 claude.ai/download 抓桌面版 App。安裝完用原本的 claude.ai 帳號登入即可，不需要另外付費。

2

切換到 Cowork 模式

Claude Desktop 左上方有模式切換器，點「Cowork」標籤進入代理模式。介面會變成左邊對話、右邊計畫與執行記錄，跟 Chat 明顯不同。

3

用白話描述整個任務

不要拆成小步驟，直接說終態。例如：「整理 Downloads 資料夾，把超過 30 天的圖片按日期分資料夾，影片類的單獨放一個 archive」。Claude 自己會規劃順序。

4

先看計畫再按 Approve

Claude 給出行動計畫後，一定要先讀過再同意。代理模式能真的動你的檔案和 App，第一次用請用低風險任務練手，例如「整理桌面截圖」，不要直接讓它動工作檔案。

5

到 Connectors 連第一個外部工具

左側 Connectors 面板，建議先連 Google Drive 或 Notion。連好後 Claude 可以直接讀你的文件、建立頁面、更新表格。完整清單包含 Gmail、DocuSign、FactSet、Slack、Microsoft 365 等 38+ 個整合。

⚡ 第一次用必做的一件事

建立 ~/Claude-Workspace/ 資料夾，放 3 份 context 檔

在家目錄開一個 Claude-Workspace 資料夾，裡面放三份 Markdown：about-me.md（你是誰、工作目標、成功標準）、brand-voice.md（寫作風格範例、禁忌詞）、working-preferences.md（行為規則、不要做什麼）。每次開新任務第一句話告訴 Claude「先讀這個資料夾」。有沒有這三份檔案，輸出品質差異非常大。

🚀進階玩法：Elijah 實戰 workflow

Notion MCP 整合，對話中直接更新頁面

用在：筆記整理、專案追蹤、CRM 維護

裝 Notion Connector 後，在 Cowork 對話裡直接說「把這份會議紀錄摘要更新到 Notion 頁面 [URL]」，Claude 直接呼叫 API 完成，不需要開瀏覽器貼上。整合 Notion 後，Cowork 就變成「會自己整理進資料庫的秘書」。

Scheduled Tasks 排程晨間日報

用在：固定重複性工作自動化

設定一次永遠跑：「每天早上 8:00 摘要 Gmail 未讀、整理今日行事曆、查今日 3 個關鍵客戶的最新動態，輸出 Daily Briefing 到桌面 briefing.md」。起床時日報已經躺在桌面，不用再手動點開一堆 App。

💡 注意：排程只在 Claude Desktop 開啟時執行，電腦休眠會跳過（但會補記錄，下次開機可以補跑）。

Sub-agents 平行處理大量工作

用在：批次研究、競品分析、多檔案處理

任務描述裡說「分析這 20 個競品的官網和定價頁」，Cowork 會自動派 5 個 sub-agents 各處理 4 個，30 分鐘的工作壓縮到 4 到 5 分鐘完成。sub-agents 有獨立 context，不會互相污染主對話。

Projects 隔離上下文，多客戶不打架

用在：代理商、Freelancer、多專案並行

行銷工作一個 Project、財務分析另一個、個人寫作再一個。每個 Project 獨立記憶、獨立 Connectors、獨立 Instructions。客戶 A 的品牌 voice 不會污染客戶 B 的報告，這是 Cowork 最被低估的功能。

Computer Use 讓 Claude 操作你的 App

用在：沒有 API 的老軟體、重複性點擊工作

macOS 用戶可開 Computer Use，Claude 能直接看你的畫面、移動滑鼠、輸入文字。實際應用：「幫我把這份 Excel 的每一行複製到公司內網的 CRM 系統」。雖然慢，但能解決 API 不存在的場景。第一次用會嚇到，之後就回不去手動了。

💡 安全建議：Computer Use 開啟時請關掉銀行、密碼管理員，避免 Claude 誤點到敏感視窗。

Dispatch 遠端派單，電腦在家背景跑

用在：出門在外但想讓電腦繼續工作

手機 Claude App 打開，描述任務並標記成 Cowork Task，指定你家電腦執行。你通勤時在手機上指派「整理本週會議逐字稿、產出摘要、寄給團隊」，回到家電腦已經跑完了。

⚠️常見踩坑 + 解法
❌ 排程任務時間到了沒執行
✅ 電腦必須保持開機、Claude Desktop 不能關、不能休眠。Windows 在電源設定禁止「顯示器關閉時睡眠」，mac 用 caffeinate 指令防睡眠。
❌ Connector 連了但授權失敗
✅ Team / Enterprise 方案需管理員批准，個人方案試重新 OAuth。授權期限一般 60 天，過期會靜默斷線。
❌ 每次輸出風格都不一樣，品質不穩
✅ 沒建 context 檔的通病。一定要寫 brand-voice.md 丟進 Workspace，並在首句告訴 Claude 先讀。
❌ 任務跑到一半中斷
✅ 多半是電腦睡眠導致。設定電源為「永不睡眠」或在任務開始前關閉其他會搶資源的 App。
❌ 把步驟寫太細，Claude 被綁手綁腳
✅ 描述終態就好，不要寫執行步驟。說「按專案分類所有 Downloads 檔案」比「先列檔案、再按日期排序、再移到資料夾」有效得多。
