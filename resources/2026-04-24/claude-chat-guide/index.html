10 分鐘從註冊到上手

最基礎的 Claude 入口，不用會寫程式，白話問問題就能讓 AI 幫你做事

💡這是什麼，跟 ChatGPT 差在哪
Claude Chat 就是 claude.ai 上的對話介面，Anthropic 出的 AI 聊天工具。跟 ChatGPT 最大的差別不是「誰比較聰明」，而是 Claude 特別強在長文本處理、寫作不油膩、跟程式碼理解。如果你常常要分析文件、寫東西、或需要 AI 回得像個「有判斷力的同事」而不是「討好型客服」，Claude 會更順手。
🌱基礎上手：5 步驟完成設定
前置條件：有 Google 或 Apple 帳號即可註冊，免費方案就能用。進階功能（記憶、Projects、擴充額度）需 Pro 方案 $20/月，重度使用者建議直接上 Max $100/月。
1

到 claude.ai 用 Google 一鍵登入

不用填信用卡、不用驗證電話，兩秒完成註冊。首次登入會問你想用什麼名字、工作類型，照實填即可，之後 Claude 會根據這個微調回答風格。

2

直接白話問，不要背「提示詞技巧」

把 Claude 當成剛到職的聰明同事：告訴它你要什麼、背景是什麼、希望格式長怎樣。不用學「你是一位資深 xxx 專家」那套，那是 2024 年的老招了。

3

點左側「Projects」建立第一個 Project

Projects 是 Claude 最重要的功能。每個 Project 有獨立的記憶、獨立的背景文件、獨立的對話歷史。行銷工作開一個 Project，財務分析開另一個，兩邊假設互不污染。

4

在 Project 的 Instructions 寫入身份＋偏好

這個欄位等於「每次對話自動帶入的系統提示」。建議寫三件事：你是誰、希望 Claude 用什麼口吻、有哪些禁忌（例如不要用 bullet、不要用 em dash、回答用繁體中文）。

5

開啟 Memory 讓 Claude 跨對話記住你

Settings > Memory 確認功能已啟用。Memory 會自動記錄你多次提過的偏好、常用工作模式，下次在別的對話也會帶入。這是 Claude 從「工具」變「助手」的關鍵差別。

⚡ 第一次用必做的一件事

先開 Project 再聊，不要在 New Chat 裡裸聊

這是新手最常浪費時間的地方。每次開 New Chat 都要重說一次背景，半小時後翻回來什麼都記不住。先花 3 分鐘開 Project、填 Instructions，之後每次對話都直接進入狀況，省下的時間以週為單位。

🚀進階玩法：Elijah 實戰 workflow

XML 結構化提示，輸出穩定度翻倍

用在：複雜任務、多條件的寫作或分析

把提示用 XML 標籤分區，Claude 的解析準確度會明顯提升。例如：<context>...</context>、<task>...</task>、<constraints>...</constraints>、<examples>...</examples>。這是 Anthropic 官方文件推薦的寫法，寫起來麻煩但一次就對。

💡 為什麼有效：Claude 的訓練資料裡 XML 結構化內容比純文字多，所以它對標籤邊界的理解天生比較準。

蘇格拉底式提問法：讓 Claude 先反問再做事

用在：想做但需求還不清楚的任務

不要直接下命令，改說：「我想達成 [目標]，但我自己也還沒想清楚細節。你先讀我貼的背景資料，然後問我 5 個最關鍵的問題，幫我把需求釐清再動手。」這樣 Claude 會先幫你整理思路，最後做出的東西才真的貼近你想要的。

三層記憶架構：不同身份不互相污染

用在：身兼多職、多客戶、多專案

第一層 Settings 的 Global Memory，放跨專案都適用的個人偏好（語言、寫作風格、禁忌）。第二層 Project Instructions，放該專案獨有的背景（客戶資料、品牌 voice、KPI）。第三層 對話開頭的臨時指令，放單次任務的特殊需求。三層分開，假設不打架。

多文件並行分析：一次丟 10 份文件比逐份問快 7 倍

用在：市場研究、競品分析、大量資料整理

把文件全部上傳到 Project Files（不是貼進對話框），然後下指令：「比對這 10 份文件，找出所有作者都同意的 3 個論點，和 2 個明顯分歧的地方。」Claude 會跨文件引用，速度比逐份問快非常多。文件放 Project Files 而不是對話框是關鍵，貼進對話框會吃掉 context。

Anti-AI-Writing-Style 清單：讓輸出不像 AI 寫的

用在：所有要給人看的寫作任務

在 Instructions 明確列禁詞：「遊戲規則改變者」「在這個快速進化的時代」「不僅…更是…」「讓我們一起探索」「值得注意的是」。再貼 2 到 3 段你自己實際寫過的文字當樣本，請 Claude 模仿。這招對輸出品質的提升，比任何「提示詞技巧」都明顯。

💡 進階：Google「anti-ai-writing-style.md」直接抄現成的禁詞清單。
⚠️常見踩坑 + 解法
❌ 每次開新對話都要重說背景
✅ 用 Projects + Instructions，把不變的背景固定下來。
❌ 輸出都像 AI 範文，讀起來很油
✅ 在 Instructions 貼 2 到 3 段自己的寫作樣本，讓 Claude 學你的語氣。
❌ 長文分析到後段品質突然下滑
✅ 對話框右上 % 超過 80% 就開新對話，把結論當交接摘要帶過去。文件也要放 Project Files 而不是貼對話框。
❌ 找不到以前聊過的內容
✅ Ctrl/Cmd + K 全局搜尋歷史對話，比手動翻側邊欄快 20 倍。
❌ Claude 回答太拘謹、不敢給意見
✅ 在 Instructions 寫「我要的是有判斷力的同事不是客服，意見請直接說，不用先鋪陳。」
