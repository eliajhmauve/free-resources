Claude Code 懶人包
CLI + MCP + Skill + Hook 完整指南

這不是「聰明的自動補全」，是能讀懂整個 codebase、跑 git、改檔案、串工具的完整開發代理

CLAUDE 四大產品系列
1. Chat 2. Cowork 3. Code 4. Design
💡這是什麼，為什麼工程師都在瘋
Claude Code 是 Anthropic 官方的 CLI 開發工具，把 Claude 塞進你的終端機，讓它能直接讀檔、改檔、跑指令、操作 git。跟 Cursor、Copilot 最大的差別是 Claude Code 的「代理能力」，它會自己規劃完整任務、呼叫工具、驗證結果。真正讓它威力爆發的，是四個延伸機制：MCP、Skills、Hooks、Subagents。只懂基礎用法大概發揮 30% 能力，搞懂這四個才算真的進入 Claude Code 的世界。
🌱基礎上手：5 步驟裝好開跑
前置條件：Node.js 18 以上、訂閱 Claude Pro $20/月（內含 Claude Code 存取）或用 API key（重度使用直接上 Max $100/月 額度最划算）。Windows、macOS、Linux 都支援，Windows 建議搭配 Git Bash 或 WSL。
1

一行指令安裝

打開終端機執行安裝指令，裝好後任何資料夾都能用 claude 啟動。

npm install -g @anthropic-ai/claude-code
2

進專案資料夾啟動

cd 到你的 repo，直接輸入 claude，它會讀取資料夾結構、先建立內部 context。第一次用不要急著下指令，先問它「解釋這個 codebase 的架構」讓它先讀過專案。

cd ~/my-project claude
3

在根目錄建立 CLAUDE.md

這是 Claude Code 最重要的檔案，等於「專案級系統提示」。內容至少要寫三件事：你的技術棧、禁止動作（例如「不要修改 config/prod.yaml」）、常用指令（npm run dev、pnpm test 等）。每次啟動 Claude 都會自動載入，不用重複交代。

4

輸入 /help 看所有指令

Claude Code 有大量內建 slash commands：/clear 清對話、/compact 壓縮 context、/model 換模型、/mcp 管理 MCP、/agents 管理 subagents。新手先熟 /help 和 /compact 兩個就夠。

5

設定永久權限，免去每次確認

編輯 ~/.claude/settings.json，加入允許清單。沒設定的話每次 Claude 要動檔案都會跳出確認，嚴重拖慢速度。新手建議先開 Read、Edit、Bash 這幾個基本權限，危險指令另外處理。

{ "permissions": { "allow": ["Read", "Edit", "Write", "Bash"] } }
⚡ 第一次用必做的一件事

CLAUDE.md 寫明「絕對不能動什麼」

這比寫「要做什麼」更重要。最低限度要禁止：生產環境設定檔（config/prod.*）、環境變數（.env）、上線部署設定（vercel.json、deploy/）、CI pipeline（.github/workflows）。沒寫這段，哪天 Claude 幫你「順手優化」把 prod.yaml 改掉就慘了。這是全世界 Claude Code 用戶第一次踩坑的 top 1。

🚀進階玩法：四大延伸機制
搞懂 MCP、Skills、Hooks、Subagents 這四個，Claude Code 才真的從「AI 工具」變成「可編程的開發代理平台」。以下每個都附實戰範例，直接抄可用。
MCP

Model Context Protocol：串接外部系統的標準

把 Notion、GitHub、Slack、Playwright 變成 Claude 的原生指令

MCP 是 Anthropic 推的開放協議，把外部工具的 API 包裝成 Claude 能直接呼叫的指令。裝一次，之後整個 Claude 生態（Code、Desktop、Cowork）都能用同一個 MCP server。

實戰範例 1：Notion MCP
claude mcp add --transport http notion https://mcp.notion.com/mcp
裝好後直接說「把這個函式的文件更新到 Notion 頁面 [URL]」，Claude 呼叫 API 自動寫入，不用開瀏覽器。
實戰範例 2：Playwright MCP
裝好後 Claude 變成自動化 QA，對話裡說「開瀏覽器測試登入流程，截圖每一步，最後寫 bug report」，它會真的開 Chromium 跑完整測試。
實戰範例 3：GitHub MCP
裝 GitHub MCP 後可以說「列出這 repo 所有 open issues，按 label 排序」、「幫我建一個 PR，title 是 xxx」，不用切出去開 GitHub 網頁。
SKILLS

Skills：可重複使用的 workflow 模板

把常用指令包裝成 /slash-command，不用每次重貼 200 字提示

Skills 就是放在 ~/.claude/skills/ 或 .claude/skills/ 的 Markdown 文件，每個文件對應一個 slash command。設計 skill 的時候，把「你希望 Claude 每次都照著做」的標準流程寫進去，之後只要打 slash 一鍵觸發。

實戰範例：/code-review skill
在 ~/.claude/skills/code-review/SKILL.md 定義檢查清單（安全漏洞、效能、命名、測試覆蓋），之後要做 review 只要打 /code-review，Claude 自動按清單掃 diff，不用重複貼 prompt。
進階：Auto-trigger Skill
在 SKILL.md 的 frontmatter 加 auto: true，Claude 偵測到特定情境會自己觸發對應 skill。例如 reuse-guard skill 在你要求寫新程式時自動先搜現有可重用的 code。
HOOKS

Hooks：事件驅動的確定性控制

在 Claude 生命週期的 12+ 個事件點掛 shell script，確定性比 LLM 高

Hooks 是在 PreToolUse、PostToolUse、SessionStart 等事件發生時自動觸發的 shell script 或 HTTP endpoint。跟 Skills 的差別是 Hooks 是「程式」不是「提示」，執行結果是確定的，適合做安全控制和自動化。

實戰範例 1：PostToolUse 自動格式化
Claude 每次寫完檔案後自動跑 prettier --write，不用每次手動格式化。
實戰範例 2：PreToolUse 攔截危險指令
寫一個 hook 偵測到 rm -rf、git push --force、DROP TABLE 就 exit code 非 0 直接擋下。比起期待 Claude 不會犯錯，這個是硬門。
實戰範例 3：SessionStart 自動補 context
每次開啟 Claude 時自動執行 git log --oneline -10 顯示最近 10 個 commit，讓 Claude 開場就知道專案近況，不用每次問。
SUBAGENTS

Subagents：平行隔離執行

獨立 context window 的迷你代理，避免主對話污染、支援平行

Subagents 是有獨立 context 的小型代理，定義在 ~/.claude/agents/。適合兩種場景：一是需要「乾淨 context」的重複性查詢（主對話已經累積太多訊息），二是需要平行執行的大量獨立任務。

實戰範例 1：研究 subagent
定義一個 researcher subagent 專門跑 WebSearch + 網頁分析。主對話要研究的時候呼叫它，大量搜尋結果只留在 subagent 的 context，主對話只拿結構化摘要，不會被雜訊塞爆。
實戰範例 2：isolation: worktree
Subagent 定義裡設 isolation: worktree，每個 subagent 操作獨立 git worktree 副本。可以派 3 個 subagent 同時改不同 feature，最後主對話決定哪個成果要合併。
實戰範例 3：代理團隊
定義 planner、coder、reviewer 三個 subagent。主對話只做任務派發，planner 先出計畫、coder 實作、reviewer 檢查，流水線化的開發體驗。
⚠️常見踩坑 + 解法
❌ CLAUDE.md 沒寫禁止清單，Claude 亂改 config
✅ 第一件事就寫「絕對不要修改以下檔案：config/prod.*、.env、.github/workflows/」。寫得越明確越好。
❌ Hook 設了但沒觸發
✅ 事件名稱必須精確匹配，Write 不等於 write。檢查 ~/.claude/settings.json 的 hooks 區塊有沒有拼錯。
❌ 裝了 10 個 MCP 結果啟動很慢
✅ 2026 起 MCP 已改為延遲載入，影響變小。但還是建議只裝當前專案會用的，用 /mcp 管理。
❌ Subagent 改的檔案結束後消失
✅ 未設 isolation: worktree 時 subagent 的暫存修改可能不保留。需要持久化的工作請開 worktree 模式。
❌ Context 爆掉輸出品質下滑
✅ 輸入 /context 查看用量，超過 70% 主動用 /compact 壓縮，不要等它自動觸發。
❌ Windows 環境變數沒生效
✅ Windows 優先用 Git Bash 或 WSL，避免 PowerShell 的路徑解析問題。Python 相關請用 python3 而非 py。
