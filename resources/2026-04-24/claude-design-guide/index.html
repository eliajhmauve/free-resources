💡這是什麼，跟 Figma / Canva 差在哪
Claude Design 不是設計軟體，是「對話式視覺稿產生器」。你用白話描述要做什麼（投資人簡報、產品一頁式、Landing Page 原型），Claude 在畫布上直接生成可互動的結果，你再透過對話調整。定位不是取代 Figma，而是「從 0 到 80 分的起點」。適合創辦人、PM、行銷人、內容創作者，不會 Figma 也能產出專業視覺稿；設計師則能用它快速打草圖，精修再進 Figma。
🌱基礎上手：5 步驟完成設定
前置條件：Claude Pro $20/月 以上方案皆可（Pro / Max / Team / Enterprise）。Enterprise 用戶需管理員在 Organization Settings 先啟用才會看到入口。可直接網頁使用，不需安裝 App。
1

前往 claude.ai/design

首次進入會觸發設計系統建立流程，不要跳過這一步。Claude 會問你要為哪個品牌建設計系統、如何匯入素材。

2

匯入品牌素材建立設計系統

有三種方式：上傳 Figma 檔案、上傳品牌 Guideline PDF、連接 codebase 讓 Claude 直接讀前端 tokens。任選一種都可以，Claude 會自動萃取主色、次色、字型、間距、圓角、元件規範。

3

左側聊天框輸入任務

描述要什麼，越具體越好。例如：「做一個給投資人看的產品 demo 一頁式，突顯核心 3 個功能，最上方是 Hero 區塊，下面是功能對比表格，最後是數據證明區」。Claude 開始在右側畫布生成。

4

用對話持續精修

生成後不滿意不用重做，直接對話調整：「Hero 文字縮小 20%」、「主色從紅換成深藍」、「第二個 section 加圓角」、「這個按鈕放右上」。每次修改即時反應在畫布上。

5

匯出成多種格式

完成後一鍵匯出：PDF 給客戶看、PPTX 給簡報用、HTML 可直接分享連結（不用帳號就能開）、Canva 給設計師繼續精修。不同場景對應不同格式，不用重做。

⚡ 第一次用必做的一件事

入門就把設計系統建好，不要跳過

沒建設計系統直接開始做案子，Claude 每次輸出風格都不同，你會花大把時間在「改配色、統一字型」的鬼打牆裡。花 15 分鐘把品牌設計系統建好，之後每個新專案自動套用，不用再重新指定配色和字型。這是 Claude Design 跟其他 AI 設計工具最大的差別，也是被低估最多的功能。

🚀進階玩法：Elijah 實戰 workflow

用網頁 URL 當參考，擷取競品視覺

用在：做 Landing Page、同行業參考

直接把競品 URL 丟進對話：「參考 linear.app 的 Hero 風格，但色系換成深藍，CTA 按鈕風格更銳利」。Claude 會擷取該網頁的視覺元素作為參考，確保原型風格符合你心目中的樣貌，省掉描述視覺細節的苦工。

Export to Claude Code：原型秒變上線程式碼

用在：PM 到工程師的交接

完成互動原型後點「Export to Claude Code」，Claude Design 會把設計稿轉成生產就緒的 React / Next.js 程式碼，工程師打開 Claude Code 直接開發，零翻譯成本。PM 畫原型 → 工程師接 code，整段流程從三天縮短到半天。

💡 關鍵心法：Design 和 Code 共用同一個 Claude 生態，這是跟 Figma + Cursor 組合最大的差別。

多品牌管理：代理商一帳號接所有客戶

用在：接多個客戶的設計師 / 代理商

一個帳號下維護多套設計系統，按專案切換。客戶 A 的品牌配色不會污染客戶 B 的提案。這點 Figma 要靠 Team Workspace 才能做到，Claude Design 原生就支援。

投資人 Pitch Deck 快速版

用在：募資、商業提案

輸入「產品一句話描述 + 目標市場 + 核心數據 + 目前進度」，Claude 生成完整 Pitch Deck 框架（問題、解決方案、市場、商業模式、團隊、要錢做什麼）。再逐頁精修，一個下午完成本來要兩週的工作。

Inline Comment 精修：畫布上點選直接標註

用在：複雜版面的局部修改

在畫布上對特定元件點右鍵加 inline comment，Claude 會針對那個元件修改，不會誤動別的地方。比在聊天框裡描述位置更精確，特別適合「這個按鈕」、「這段文字」的局部調整。

💡 如果 inline comment 沒識別，同樣的描述貼進聊天框也有效，不用卡在介面。

獨立 HTML 匯出：一個檔案寄給客戶

用在：不想給客戶裝帳號但想給他看互動原型

匯出為獨立 HTML 檔案，一個 .html 檔包含全部資源，客戶用瀏覽器開就能看可互動原型。不需要 Claude 帳號、不需要任何軟體、不需要 server。寄信附件直接丟過去，對方雙擊就能操作。

⚠️常見踩坑 + 解法
❌ 提示太模糊，輸出不對味
✅ 提示越具體越好。說「三欄式 SaaS 定價頁，第二欄 highlight，每欄 5 個 feature bullet，底部加 FAQ」比「做定價頁」有效 10 倍。
❌ 以為 Claude Design 能取代 Figma
✅ 定位是「從 0 到 80 分的起點」，不是取代專業設計工具。複雜的品牌設計、精細的互動、設計系統維護，Figma 仍然不可取代。Claude Design 適合打草圖、做原型、快速提案。
❌ Enterprise 用戶看不到入口
✅ 管理員需先到 Organization Settings 啟用 Claude Design，才會對成員顯示。個人方案不用做這件事。
❌ 設計系統建了但沒自動套用
✅ 確認是在 Organization 層級上傳品牌素材，不是個人帳號。Team / Enterprise 常見的設定錯誤。
❌ 匯出 PPT 後字型變了
✅ Claude Design 的字型若是自訂字型（例如思源黑體、Inter），匯出到 PPT 時對方電腦沒裝字型會 fallback。建議匯出 PDF 保留視覺一致性，或同時附上字型檔。
❌ 擔心做幾張設計稿就把 Chat 額度用光
✅ Claude Design 和 Chat 的用量分開計算，兩邊獨立配額，不會互相扣。重度使用設計功能不用擔心會影響日常對話額度。
