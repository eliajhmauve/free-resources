💡這是什麼，跟 DALL-E 差在哪

1

到 chat.openai.com 試 Instant 模式

免費用戶直接在 ChatGPT 對話框輸入「幫我做一張 [描述]」，會自動跑 Images 2.0 Instant 模式，3-5 秒出圖。Free tier 約每 3 小時 3-10 張動態配額。

2

判斷你需要 Instant 還是 Thinking

純藝術探索、社群快速圖卡、商品照 → Instant 就夠（快、便宜）。複雜多元素構圖、商業印刷素材、需要文字精準、需要批量系列圖一致 → 開 Thinking。Thinking 延遲 10-30 秒，成本是 Instant 2-3 倍，但品質差距明顯。

3

API 第一次跑：機械式替換 DALL-E 3

如果你之前用 DALL-E 3，遷移就是改一個參數。response 結構完全相同，b64_json / url 都不變。唯一注意：DALL-E 3 的 quality="hd" 改成 quality="high"。

from openai import OpenAI client = OpenAI() result = client.images.generate( model="gpt-image-2", # 唯一改動 prompt="...", size="1024x1024", quality="high", # hd → high n=1, )
4

寫第一個會自動精準渲染中文的 prompt

關鍵指令：用全大寫前綴 EXACT TEXT: "你要的字串"，prompt 結尾加 Render text verbatim. No extra words. 解析度拉到 1024x1024 以上、品質設 medium 或 high，中文字準確率立刻拉到 95% 以上。

A promotional banner for a coffee brand. Center: matte black ceramic cup on wood table. Top: bold sans-serif text reads "EXACT TEXT: 春季限定 - 焙煎深度烘焙" Render all text verbatim. No extra words. Size: 1920x1080
5

避免第一週最容易踩的 3 個坑

第一坑：以為 n=8 就會角色一致 → 必須加 quality_mode="thinking"。第二坑：中文長文字（>20 字一行）漏字 → 拆成多行、解析度 1536+。第三坑：Thinking 模式吃額度 → 開發階段先用 Instant + low quality 測試構圖。

⚡ 第一次用必做的一件事

立刻檢查你有沒有 DALL-E 3 整合的舊程式碼

2026/5/12 之後 DALL-E 2/3 API 完全關閉。如果你有 production 服務還在用 model="dall-e-3" 或 model="dall-e-2"，5/13 早上就會返回錯誤。今天花 30 分鐘搜尋 codebase，把 model 名稱換成 gpt-image-2，順便回歸測試一輪（GPT Image 2 比 DALL-E 3 更字面執行 prompt，部分 prompt 需要重新調整）。

🚀進階玩法：5 個實戰 workflow

批量產 8 張角色一致的 IG 系列圖

用在：知識型內容、品牌系列、Pitch Deck

關鍵組合：n=8 + quality_mode="thinking" + prompt 中對角色做「錨點描述」（same character: [外觀], face/outfit must remain identical across all images）。Thinking 模式會建內部角色 embedding，跨 8 圖維持臉、體、服裝一致。純 n=8 不加 thinking 等於 8 次獨立生成，無一致性。

💡 進階：若有參考圖，用 /v1/images/edits 端點上傳作 reference，一致性比 generations 端點更高。

讓 Claude / Cursor 自動串 GPT Image 2 產圖

用在：開發工作流、AI 產線

用 OpenAI SDK 包裝成 MCP server 或 Skill，Claude Code 或 Cursor 對話中直接說「產一張 XXX 圖」就觸發。這對「研究素材 → 自動產配圖」的工作流是 game changer（建議走 medium quality 控制成本，每張 ~$0.053）。

Web Search 整合：產出含當下事件的視覺

用在：時事新聞圖、新品包裝參考、最新設計趨勢

Thinking 模式可即時呼叫網路搜尋，抓最新視覺參考再生成。突破訓練資料截止（2025/12）的限制。例如「做一張關於 Anthropic 4/17 推出的 Claude Design 的概念圖」，模型會先 web search 抓 Claude Design 的官方視覺，再生成符合品牌的圖。

💡 限制：僅 Thinking 模式可用，Free tier 沒有此能力。

多語文字渲染（特別中日韓）

用在：海外行銷、多語社群、亞洲市場素材

過去 DALL-E 渲染日文、韓文常出現亂字、混入英文，GPT Image 2 在 CJK 字符達 95%+ 準確。實測一張圖內可同時出現「日文標語 + 英文品牌名 + 數字促銷碼」三語，互不干擾。對接亞洲市場的品牌素材立刻省去後製疊字工序。

4K 高解析度商業印刷素材

用在：實體廣告、雜誌封面、印刷品

API beta 支援 3840x2160（4K landscape）和 2160x3840（4K portrait）。4K high quality 約 $0.41/圖，比外包設計便宜兩個數量級。需要符合 Size 約束：最長邊 ≤3840、雙邊均為 16 倍數、寬高比 ≤3:1。

💡 ChatGPT 介面目前仍是 2K 上限，4K 必須走 API。
⚠️5 個常見踩坑 + 解法
❌ 文字渲染失敗（亂字、錯字、字母變形）
✅ 用 EXACT TEXT: "..." 標記、prompt 結尾加 Render verbatim. No extra words.、解析度 1024+、品質 medium 或 high。
❌ 設了 n=8 但每張臉都不同
✅ 必須加 quality_mode="thinking"（純 n=8 等於獨立生成）。Prompt 中描述「same character: [外觀]」當錨點。有參考圖優先用 edits 端點。
❌ Thinking 模式帳單突然飆高
✅ 推理 token 另計，成本是 Instant 2-3 倍。開發期先用 Instant + low quality 測試構圖，確認方向後再切 Thinking + high。每月監控 Dashboard 用量。
❌ 中文長文字（一行 >20 字）漏字
✅ 拆成多行、每行 12-15 字以內。解析度強制 1536+。或者：圖像生成不含長文字，後製在 Canva/Figma 疊上文字層（最穩做法）。
❌ DALL-E 3 遷移後 prompt 出來的圖跟以前不一樣
✅ GPT Image 2 比 DALL-E 3 更字面執行 prompt，原本 DALL-E 3 會做創意詮釋的部分要重新調整 prompt 措辭。建議遷移後對所有現有 prompt 跑一輪回歸測試。
